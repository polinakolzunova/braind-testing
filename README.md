# Выполнение тестового задания Braind

## Задание 1.
Реализовать скрипт на PHP
В переменной $articleText хранится текст статьи. В переменной $articleLink хранится ссылка на страницу с полным текстом этой статьи.

Нужно в переменную $articlePreview записать анонс статьи (сокращённый текст) по правилам:
* обрезать до 200 символов;
* приписать многоточие;
* последние 3 слова и многоточие сделать ссылкой на полный текст статьи.

Дополнительно опиши текстом какие проблемы ты видишь в решении этой задачи? Что может пойти не так?

Результат: ссылка на репозиторий с кодом и твой комментарии.

## Задание 2.
Реализовать выборку из БД (MySQL, PHP).

Структура БД:
Необходимо создать 3 таблицы:
* Товары (id, Название, Категория, Цена).
* Свойства товаров (id, Название свойства).
* Значение свойств товаров (id, id_Товара, id_Свойства, Значение).
Должны быть указаны все первичные ключи, автоинкремент и прочее.

Задача:
Есть набор свойств товара — цвет, размер, новинка. Необходимо выбрать все новинки зеленого цвета и вывести их на странице (верстка не имеет значения, просто в читаемом виде). В списке должна быть информация о размере и цвете товара. Необходимо реализовать механизм постраничной навигации по отфильтрованным товарам, по 3 товара на странице. Добавить 10 товаров для тестирования. Не должно быть запросов в цикле, рекомендуется использовать JOIN.

Результат: ссылка на репозиторий с дампом БД и php-скриптом.

## Задание 3.
Интерфейс добавления и просмотра данных.

В таблице БД хранятся города. Нужно создать интерфейс добавления нового города в таблицу и просмотра списка городов из таблицы с использованием HTML, CSS, PHP, возможно JS.
Структуру таблицы придумайте самостоятельно. Удаление и редактирование делать не нужно.

Форма решения произвольная, преимущество будет у решений, в которых будет присутствовать:
* защита от инъекций при получении параметров через GET или POST;
* обработка текста, вводимого пользователем (вырезание тегов, экранирование и т.п.);
* осмысленное название полей и таблицы базы данных;
* разумное оформление кода: имена переменных, функций, отступы, комментарии;
* отсутствие лишнего кода (бездумный копипаст решений со stackoverflow);
* разделение логики работы и представления (MVC, разделение на шаблон и логическую часть, например);

Визуальное оформление не принципиально, можно воспользоваться HTML/CSS-фреймворками.

PHP-код надо написать с нуля.
Результат: ссылка на репозиторий с кодом.
